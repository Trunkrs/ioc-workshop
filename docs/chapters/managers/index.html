4. Implement business rules

In the first chapter, we’ve looked at the processes and defined our business rules. In this chapter, we’re going to use those definitions to set up our business layer. This layer consists of managers that implement the business rules and constraints. It will contain most of the important decisions in our code. It also delegates commands to repositories and clients on the data and service layers.

The manager#

Let’s look again at the business rules we’ve defined in our first chapter. They are:

  • When the customer selects a drink and the drink is available then show the price otherwise ask the customer to select another drink
  • When the customer selects a drink and the drink is available then activate the payment terminal
  • When the bank has approved the transaction then dispense the drink
  • When the bank rejects the transaction then go back into the initial state

The next step is defining the dependencies of our manager. From our expectations we can see three clear dependencies:

  1. DrinkRepository: The repository that interacts with the internal data storage of the vending machine.
  2. MechatronicsClient: The pre-defined client that interacts with the mechatronics of the machine. Think dispense a drink or reset the interface.
  3. PaymentClient: The pre-defined bank transaction client.

The workshop project has some pre-defined classes. These have been added to make you focus on the core values of the workshop and not having to focus on any of the parts that are not relevant.

The only dependency we have to define ourselves is the repository and connect it to the internal data store. Let’s create a stub with the appropriate models so that our manager can interact with the data store.

Hell yeah! These are our tests that define our business rules perfectly for the vending machine. Let’s now go ahead and implement our manager. This part is going to be easy!

class VendingMachineManager {  public constructor(    private readonly repo: DrinkRepository,    private readonly mechaClient: MechatronicsClient,    private readonly paymentClient: PaymentClient,  ) {}
  /**   * Executes a transaction to buy and dispense a drink.   * @param drinkNr The drink button number to dispense.   */  public async buyDrink(drinkNr: number): Promise<void> {    const drink = await this.repo.getDrink(drinkNr)    if (!drink.isAvailable) {      await this.mechaClient.resetTransaction()      return    }        const paymentResult = await this.paymentClient.doPayment(drink.price)    if (paymentResult === PaymentResult.Success) {      await this.mechaClient.dispense(drinkNr)    } else {      await this.mechaClient.resetTransaction()    }  }}
